# Security Policy

## Supported versions

Security fixes are applied to the `main` branch.

## Reporting a vulnerability

Please report vulnerabilities privately via email:
- omarelkhali@gmail.com

Include:
- affected component/file
- reproduction steps
- impact assessment
- suggested mitigation (optional)

## Secrets handling

- Never commit `LI_AT` cookies or credentials.
- Keep generated datasets out of git history unless anonymized.
